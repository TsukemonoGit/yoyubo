# yoyuboo

家計の「普通じゃない月」を記録するアプリ。

## デザイン思想

- 正確さよりも傾向の把握を重視
- シンプルな記録で家計の浮き沈みを可視化

## 主な機能

- 家計の支出・収入を万円単位で記録
- グラフで月別の変化を視覚化（未入力の月は直線補完して表示）
- バックアップ・リストア機能

## サポートプラットフォーム

| プラットフォーム | ストレージ                         | 状態     |
| ---------------- | ---------------------------------- | -------- |
| Android          | Storage Access Framework (SAF)     | 対応     |
| iOS              | ファイル選択                       | 対応予定 |
| Web              | File System Access API + IndexedDB | 対応     |
| Desktop          | -                                  | 対応予定 |

## データの取り扱い

保存ファイルの場所は任意です。適当なフォルダに作って大丈夫ですが、Google Driveなどのクラウド共有フォルダ内にファイルを作成すると、複数端末でデータを同期できます。

## 技術スタック

- **フレームワーク**: Flutter
- **状態管理**: Provider パターン
- **データ保存**:
  - Android: SAF URIs
  - Web: File System Access API, IndexedDB, localStorage
  - バックアップ: shared_preferences

## データ仕様

- 金額単位は固定（万円）
- 小数点許容（例: 142.5）
- balance が null の月は「残高未入力」として扱い、グラフでは直線補完表示

## よくある質問（FAQ）

### 金額の単位は？

「万円」固定です。142 は 142万円、142.5 は 142.5万円（142万5千円）として扱われます。正確さより傾向把握を目的としているため、大まかな値で結構です。

### balance（残高）が null の月はデータがないのか？

いいえ、「残高未入力」です。出来事メモだけが記録されている月や、まだ通帳を確認していない月でもデータとして保持しています。グラフでは null の月を前後の値で直線で結んで表示します。

### Android で OneDrive を使いたい

Android では「新規ファイルを作成」で OneDrive が選択できない仕様になっています。代わりに「既存ファイルを開く」から OneDrive 上の既存ファイルを選ぶことで利用可能です。Google Drive や Dropbox は新規作成でも利用できます。

### Web で更新したデータが Android で即時反映されない

Web と Android で同じクラウドフォルダ（Google Drive / OneDrive 等）のファイルを使っている場合、Android でファイルを開くと古いデータが表示されることがあります。数秒〜数分待つか、「既存ファイルを開く」からもう一度ファイルを選択すると最新になります。

これは SAF（Storage Access Framework）がクラウドストレージのプロバイダを使う際、ローカルキャッシュを返す仕様によるものです。コード側で回避することはできません。

### Android → Web は即時反映される

Android で更新したデータは Web で即時確認できます。キャッシュの同期は Web → Android の一方通行になります。

### 複数端末で同じファイルを使う場合の注意

クラウドストレージ上のファイルを Android と Web で共用することはできますが、上記のキャッシュ遅延が発生する可能性があります。同時編集は避けてください。
